# tarea_17
La tarea 17 consistía en realizar una serie de pasos para aprender a guardar y proteger proyectos que requieren de construirse en línea. Como es un inicio del conocimiento, se hizo una simulación para esto se comenzó utilizando Git desde mi propio computador creando un repositorio local que tenía un archivo html y un README creando el primer commit que sirve para rastrear modificaciones y ubicar donde fueron realizadas. Luego el segundo paso es subir ambos archivos al programa GitHub el cual es online permitiendo tener guardado los documentos en una nube para ser modificados por el mismo autor y por el grupo autor. Aquí se sube un segundo commit el cual rastrea los cambios realizados en GitHub. El tercer paso era construir una rama la cual es para trabajar de manera independiente del proyecto principal el cual (supongo) es el archivo html y README (en mi caso: master) se hizo modificaciones del archivo html agregando un encabezado básico como barra de navegación y se agregó un nuevo commit que describe lo que acabo de escribir. Cuarto paso -el más complejo de entender- se hizo una fusión de rama a la rama principal. Mi problema fue que mi rama principal era Master no main por lo que tuve que crear Main luego pasar de feature/navbar donde estaba la modificación de hmtl a empujar a este nuevo Main y por ultimo fusionar esto y subir a GitHub con el pull request. El quinto paso era crear el Pull Request en GitHub y luego observar las modificaciones en paralelo habia que agregar al html un <footer> para que se viera esta nueva modificación en la página y si se ve, se acepta y queda en la plataforma.
